# AI Agent Workflow

个人 AI Agent 工作流配置包，用来在不同电脑之间迁移：

- 统一提示词入口：`ai-prompt/`
- Skill/MCP 索引：`ai-prompt/capabilities/`
- 本地运行时 skills：`ai-skills/`
- MCP 配置模板：`ai-config/`
- 原生入口说明：`entrypoints/`
- 安装脚本：`scripts/install.sh`、`scripts/install.ps1`

## What Is Included

- 轻量公共提示词：中文输出、最小改动、真实验证、必要澄清。
- 统一 router：各 AI 运行时只需先读 `~/.ai-prompt/router.md`，再由它进入完整工作流。
- 高级模型路由：精确描述直接读文件；模糊/跨文件问题先判断是否需要语义/符号召回或侦查。
- MCP 索引：ADO、Chrome DevTools、Node REPL、Serena、插件/连接器能力说明。
- Engineering skills：
  - `backend-business-safety`
  - `diagnose`
  - `grill-me`
  - `improve-codebase-architecture`
  - `receiving-code-review`
  - `security-best-practices`
  - `tdd`
  - `verification-before-completion`
- Prompt skills：
  - `anysearch`
  - `bilibili-auto-transcript`

## What Is Not Included

- API keys and `.env` files.
- Machine-specific full agent runtime config.
- Plugin/cache directories generated by agent tools.
- Trust lists for local projects.
- Runtime-generated files such as `.DS_Store`, `runtime.conf`, and `__pycache__`.

## Install On A New Machine

Linux / macOS / WSL:

```bash
git clone git@github.com:egoist74110/ai-agent-workflow.git
cd ai-agent-workflow
bash scripts/install.sh
```

Windows PowerShell:

```powershell
git clone git@github.com:egoist74110/ai-agent-workflow.git
cd ai-agent-workflow
powershell -ExecutionPolicy Bypass -File scripts/install.ps1
```

The installer copies:

- `ai-prompt/` to `~/.ai-prompt`
- `ai-skills/` to `${AI_AGENT_SKILLS_DIR:-~/.ai-agent/skills}`

It also replaces `__HOME__` placeholders with the current machine's home directory.

If your AI runtime expects skills somewhere else, set `AI_AGENT_SKILLS_DIR` before running the installer.

After deployment, the installer asks two local setup questions:

- Which MCP snippets to prepare: choose `serena`, `chrome-devtools`, `ado-work-items`, `all`, or none. The installer writes selected snippets to `~/.ai-agent/mcp.selected.toml`; it does not overwrite any runtime MCP config.
- Which AI runtimes to connect: choose `claude`, `codex`, `agy`, `custom`, `all`, or none. For known runtimes, the installer suggests common native entrypoint paths such as `~/.claude/CLAUDE.md`, `~/.codex/AGENTS.md`, and `~/.agy/AGENTS.md`. Each selected file is backed up and replaced with a thin pointer to `~/.ai-prompt/router.md`.

For non-interactive installs:

```bash
AI_AGENT_MCP_SELECTIONS=serena,chrome-devtools \
AI_AGENT_RUNTIMES=claude,codex \
AI_AGENT_ENTRYPOINTS="$HOME/.some-ai/INSTRUCTIONS.md;$HOME/.another-ai/AGENTS.md" \
bash scripts/install.sh
```

## Daily Workflow

Treat this repository as the source of truth.

Edit files here:

- `ai-prompt/`
- `ai-skills/`
- `ai-config/`

Then apply the project version to global runtime directories:

Linux / macOS / WSL:

```bash
cd ~/ai-agent-workflow
bash scripts/apply-to-global.sh
```

Windows PowerShell:

```powershell
cd ~/ai-agent-workflow
powershell -ExecutionPolicy Bypass -File scripts/apply-to-global.ps1
```

The apply script copies:

- `ai-prompt/` to `~/.ai-prompt`
- `ai-skills/` to `${AI_AGENT_SKILLS_DIR:-~/.ai-agent/skills}`

It backs up the overwritten global files under:

```text
~/.ai-agent-workflow-backups/<timestamp>/
```

It does not overwrite runtime MCP config files.

After confirming the global copy works, commit and push:

```bash
git status
git add .
git commit -m "Update workflow"
git push
```

## MCP Setup Notes

The installer does not overwrite runtime MCP config files.

MCP tools are not installed or enabled automatically. Before adding a new MCP server, the agent must explain why it is needed, what command/config it will add, and wait for the user to confirm.

Use `ai-config/mcp.example.toml` as a merge reference for runtimes that use compatible MCP config. Important entries:

- `serena`: semantic/symbol code navigation.
- `chrome-devtools`: browser debugging through Chrome DevTools MCP.
- `adoWorkItems`: local personal ADO tool; path must be adjusted if the script is not present on the new machine.

For Serena:

Linux / macOS / WSL example:

```bash
uv tool install -p 3.13 serena-agent
# Then add the MCP server with your agent runtime's MCP command.
serena start-mcp-server --context=<runtime> --project-from-cwd
```

Windows PowerShell example:

```powershell
uv tool install -p 3.13 serena-agent
# Then add the MCP server with your agent runtime's MCP command.
serena start-mcp-server --context=<runtime> --project-from-cwd
```

Only run these after user confirmation. Then adjust generated commands to use absolute paths if the agent runtime cannot find `serena`, `uv`, or `uvx`.

## AGENTS.md Router

AI tools do not automatically know this workflow exists. They need a native entrypoint file or global setting that points to the prompt router.

The canonical entrypoint is:

```text
~/.ai-prompt/router.md
```

Use `AGENTS.example.md` as an entrypoint template for compatible agents. `entrypoints/README.md` describes the same thin-entrypoint pattern for other runtimes.

For any runtime, create or update the native instruction file that runtime already reads and keep it thin:

```text
Read ~/.ai-prompt/router.md first, then follow it.
```

This means other AI tools do not read this project. They read their own private/native instruction file, and that file points to the deployed shared router under `~/.ai-prompt`.

The installer can create those private/native pointer files. It offers common defaults for known runtimes, but the user can override every path because runtime private directories differ by tool, version, and machine.

## Sync Back From This Machine

When updating this repository from the current machine, copy only safe files:

- include prompt files and skill documents/scripts
- exclude `.env`, `runtime.conf`, plugin caches, and full machine configs
- replace absolute home paths with `__HOME__` before committing
